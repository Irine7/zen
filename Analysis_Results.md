## 🎯 Предлагаемые Вехи Развития (Milestones)

Так как твоя цель — **глубокое обучение бэкенду (GraphQL + Apollo + Prisma)** и написание кода своими руками, мы разобьем оставшиеся задачи на 5 образовательных вех. Каждая веха учит конкретному паттерну.

---

### 🟢 Веха 1: Решение проблемы N+1 с помощью DataLoader (Backend Deep-Dive)
*   **Почему это важно**: Сейчас, когда загружается сад, GraphQL делает один запрос на получение всех деревьев. Затем, для каждого дерева срабатывают `FieldResolver`'ы для полей `user` и `habit`. Если у пользователя 10 деревьев, сервер сделает `1 + 10 (user) + 10 (habit) = 21` запрос к базе данных. Это классическая проблема N+1.
*   **Чему ты научишься**: Батчингу (группировке) и кэшированию запросов с помощью библиотеки `dataloader`.
*   **Твоя задача**: Написать DataLoader для получения привычек и пользователей, интегрировать его в контекст Apollo Server и переписать резолверы.

---

### 🟢 Веха 2: Профили Пользователей и Статистика (Fullstack)
*   **Почему это важно**: У нас есть система очков, но нет понятия «уровня пользователя» или «статистики выращенных садов». Профиль сделает игру более вовлекающей.
*   **Чему ты научишься**: Сложным агрегационным запросам в Prisma (группировка, подсчет связанных записей) и созданию новых страниц/компонентов в Next.js.
*   **Твоя задача**:
    *   Создать Query `getUserProfile` (или дополнить текущую), которая считает: сколько всего деревьев посажено, сколько из них живые, сколько погибло.
    *   Создать экран профиля на фронтенде с красивым Glassmorphism UI.

---

### 🟢 Веха 3: Dashboard привычек «План на сегодня» (Frontend + Backend)
*   **Почему это важно**: Сейчас привычки можно отмечать только на карточке конкретного дерева. Нам нужен удобный ежедневный чек-лист (To-Do список), где пользователь видит все свои привычки, сгруппированные по статусу выполнения за сегодня.
*   **Чему ты научишься**: Работа с датами на бэкенде (сравнение дней с учетом часовых поясов), проектирование удобного пользовательского интерфейса.
*   **Твоя задача**:
    *   Добавить на бэкенд логику разделения привычек на «выполненные сегодня» и «оставшиеся».
    *   Создать боковую панель или вкладку с ежедневными задачами на фронтенде.

---

### 🟢 Веха 4: Асинхронные задачи (Cron Jobs) для проверки здоровья (Backend)
*   **Почему это важно**: Сейчас статус дерева рассчитывается только при обращении к API. Но если пользователь не заходит на сайт, база данных не знает о том, что дерево засохло. Нам нужен фоновый процесс, который будет раз в день проверять деревья и, например, присылать логи или отправлять уведомления на почту, если дерево заболело.
*   **Чему ты научишься**: Работе с асинхронностью, планировщиком задач (`node-cron` или альтернативы) и фоновыми скриптами в Node.js.
*   **Твоя задача**: Написать фоновую службу, которая запускается по расписанию и проверяет здоровье садов.

---

### 🟢 Веха 5: Визуализация, Сетка Сада и Анимации (Frontend Design)
*   **Почему это важно**: Настоящий Zen Garden должен выглядеть интерактивно. Вместо простого списка карточек деревьев мы можем сделать сетку, напоминающую реальный сад, и добавить анимации роста.
*   **Чему ты научишься**: Сложной верстке CSS Grid, работе с Framer Motion для анимации изменения состояний компонентов.
*   **Твоя задача**:
    *   Заменить список на интерактивную сетку.
    *   Добавить красивые спрайты для деревьев разного возраста (Росток -> Молодое дерево -> Взрослое -> Цветущее).
    *   Анимировать переход между уровнями.
