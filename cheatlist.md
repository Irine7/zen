GRAPHQL

Schema - описание всех данных, которые есть в приложении.
Type - тип данных. Например, Товар с полями: id, name, price.
Resolver - функция, которая обрабатывает запрос. Например, получить список всех товаров.

Query (аналог GET запроса) - запрос на получение данных. Например, получить список всех товаров.
Mutation (аналог POST и PUT запросов) - запрос на изменение данных. Например, добавить новый товар.
Subscription (аналог WebSockets) - запрос на получение данных в реальном времени. Например, получить список всех товаров.


Resolver — это мозг GraphQL API, который знает, как отвечать на запросы фронтенда
@Resolver() — помечает класс как тот, кто умеет «решать» запросы. Это функция, которая возвращает данные для определённого поля
@ObjectType — помечает класс как GraphQL-объект
@Field() — помечает поле объекта
@Query() — пометка метода, который просто отдает данные из базы
@Arg() — пометка аргумента запроса
@Mutation() — пометка метода, который изменяет данные
@FieldResolver status: Это «виртуальное» поле. Его нет в базе данных, оно вычисляется на лету
@Root() — пометка корневого объекта (в данном случае дерева)
@Ctx() — контекст запроса. Это объект, который содержит информацию о запросе
