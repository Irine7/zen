Привет! В рамках этого модуля мы создадим полноценную систему экономики для **ZenGarden**. Это не просто добавление кода — это отличная возможность освоить продвинутые и критически важные темы бэкенд-разработки: **транзакции баз данных**, **сложные связи в ORM** и **безопасность GraphQL мутаций**.

Этот план разбит на **5 последовательных шагов (уроков)**. Мы будем двигаться строго по очереди:
1. Я объясняю теоретическую часть шага и даю темы для изучения.
2. Ты изучаешь материалы, задаешь вопросы, если что-то непонятно.
3. Я даю тебе конкретное практическое задание с инструкциями.
4. Ты пишешь код, мы его проверяем, и только после этого переходим к следующему шагу!

## 🗺️ Карта нашего пути (5 Шагов)

### 📂 Шаг 1: Обновление Базы Данных (Prisma Schema & Migrations)
**Цель:** Спроектировать и создать таблицы для магазина семян и инвентаря пользователей.
*   **Теория для изучения:** Отношения "Один-ко-многим" (One-to-Many) и "Многие-ко-многим" (Many-to-Many) через связующие таблицы в Prisma. Уникальные составные индексы (`@@unique`).
*   **Что сделаем:** 
    *   Создадим модель `Seed` (доступные семена в магазине: Сакура, Бамбук, Сосна, Клен).
    *   Создадим модель `InventoryItem` (инвентарь пользователя, где хранится количество купленных семян).
    *   Запустим Prisma-миграцию, чтобы применить изменения в PostgreSQL.

### ⚡ Шаг 2: Наполнение Магазина (Database Seeding)
**Цель:** Научиться автоматически наполнять базу данных начальными (seed) данными.
*   **Теория для изучения:** Зачем нужен seeding базы данных, как написать скрипт инициализации данных в Prisma.
*   **Что сделаем:** 
    *   Напишем скрипт, который при запуске автоматически добавляет в таблицу `Seed` 4 типа семян с их ценами, названиями и описаниями.
    *   Запустим этот скрипт и проверим базу в **Prisma Studio**.

### 🔍 Шаг 3: GraphQL Queries — Витрина Магазина и Инвентарь
**Цель:** Создать GraphQL схемы и Queries, чтобы фронтенд мог запрашивать товары и инвентарь.
*   **Теория для изучения:** Декораторы `@ObjectType()`, `@Field()` в TypeGraphQL. Создание Query-резолверов с проверкой авторизации (`@Ctx() ctx`).
*   **Что сделаем:**
    *   Напишем GraphQL-сущности (Entities) для `Seed` и `InventoryItem`.
    *   Создадим запрос `getSeeds` (показывает все семена в магазине).
    *   Создадим запрос `getInventory` (показывает купленные семена текущего авторизованного пользователя).

### 💳 Шаг 4: Мутация покупки и Prisma Transactions (Самая важная часть!)
**Цель:** Реализовать абсолютно безопасную покупку семечка с проверкой баланса очков.
*   **Теория для изучения:** Что такое транзакции в БД, свойство **ACID (Atomicity)**. Почему обычные последовательные запросы к БД опасны в финансовых операциях (проблема race conditions). Как использовать `prisma.$transaction`.
*   **Что сделаем:**
    *   Создадим GraphQL мутацию `buySeed(seedId: String!)`.
    *   Напишем логику покупки внутри `prisma.$transaction`:
        1. Проверяем, авторизован ли пользователь.
        2. Ищем семечко в базе и проверяем баланс Zen Points пользователя.
        3. Если очков хватает — списываем очки у `User` и добавляем/увеличиваем количество (`quantity`) этого семени в `InventoryItem`.
        4. Если очков мало или семечка не существует — транзакция прерывается (rollback), и баланс пользователя остается в безопасности.

### 🌱 Шаг 5: Посадка дерева из инвентаря
**Цель:** Переписать/создать логику посадки Bonsai так, чтобы пользователь мог сажать только те семена, которые реально есть у него в инвентаре.
*   **Теория для изучения:** GraphQL Input Types, логика списания ресурсов из инвентаря.
*   **Что сделаем:**
    *   Создадим мутацию `plantBonsaiFromInventory(seedId: String!, habitId: String!)`.
    *   При вызове:
        1. Проверяем, есть ли у пользователя в инвентаре семечко с `seedId` (количество > 0).
        2. Уменьшаем количество на 1 (если стало 0 — удаляем запись из инвентаря).
        3. Создаем новое дерево `Bonsai` выбранного типа, привязанное к привычке `habitId` и пользователю.
