# ZenGarden: Bonsai Habit Tracker — План Развития 🪴

Этот проект превращает рутинное отслеживание привычек в медитативный сад. Каждое дерево — это реальный навык или привычка пользователя.

---

## 1. Бэкенд (Node.js + GraphQL + Prisma) 🧠
*Цель: Попрактиковаться в сложных схемах, асинхронности и БД.*

### Модуль Пользователей (User & Auth)
- [ ] **JWT Авторизация**: Регистрация и логин.
- [ ] **Система Валюты**: "Zen Points" — начисляются за выполнение привычек, тратятся в магазине.
- [ ] **Профили**: Уровни пользователя и статистика (сколько деревьев выращено).

### Модуль Привычек (Habits)
- [✅] **Связь 1-к-1**: Привязка конкретной привычки (например, "Reading English") к конкретному Bonsai.
- [✅] **Логика роста**: При отметке привычки как "выполнено" (Mutation `completeHabit`), дерево получает опыт (XP) и поливается.
- [✅] **Штрафы**: Если привычка не отмечена в течение дня, дерево переходит в статус `SICK`.

### Экономика и Магазин (Shop & Inventory)
- [ ] **Семена (Seeds)**: Разные типы деревьев (Pine, Sakura, Bamboo). Нужна таблица `Seed` в БД.
- [ ] **Инвентарь**: Таблица для хранения купленных, но еще не посаженных семян и удобрений.
- [ ] **Транзакции**: Покупка предметов (Mutation `purchaseItem`) с проверкой баланса Zen Points через Prisma Transaction.

### Продвинутый GraphQL
- [ ] **Сложная фильтрация**: Запрос `getGarden` с аргументами (filter by type, status, level).
- [ ] **Subscriptions**: Реал-тайм уведомления, когда дерево достигает нового уровня или засыхает.
- [ ] **Dataloaders**: Оптимизация запросов, чтобы избежать проблемы N+1 при получении привычек для всех деревьев в саду.

- [ ] **Анимации**: Плавный рост дерева при левел-апе (используя Framer Motion).

## 3. Идеи для Символизма 🏮
Каждое дерево в японской культуре имеет значение:
- **Сосна (Pine)**: Долголетие и стойкость (для спорта/здоровья).
- **Сакура (Cherry)**: Красота и быстротечность (для творчества/хобби).
- **Клен (Maple)**: Спокойствие и мудрость (для медитации/обучения).
- **Бамбук (Bamboo)**: Гибкость (для йоги/растяжки).

## 4. Продвинутые Технологии (Deep Dive) 🚀
*Здесь собраны задачи для прокачки навыков, о которых ты просила:*

### GraphQL & API
- [ ] **DataLoader**: Решение проблемы N+1. Когда мы запрашиваем список всех деревьев и для каждого хотим подтянуть детали привычки, DataLoader поможет сделать это одним запросом к базе вместо 100.
- [ ] **GraphQL Subscriptions**: Использование WebSocket для обновления состояния дерева на фронте в реальном времени (например, когда проходит 24 часа и статус меняется на `SICK`).
- [ ] **Custom Directives**: Например, директива `@auth`, которая автоматически проверяет права доступа к мутации.

### Backend & DB
- [ ] **Asynchronicity (Асинхронность)**: 
  - Реализация фоновых задач (Cron Jobs) для проверки здоровья всех деревьев раз в сутки.
  - Интеграция с внешними API (например, погода, которая влияет на скорость роста).
- [ ] **Prisma Transactions**: Безопасная покупка в магазине — списание денег и добавление предмета должны происходить либо оба сразу, либо ни одного (атомарность).
- [ ] **Middleware**: Логирование всех запросов и обработка ошибок.
